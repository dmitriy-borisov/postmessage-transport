[postmessage-transport Code of Conduct](CODE_OF_CONDUCT.md).

to the project maintainer.

## I Have a Question

> If you want to ask a question, we assume that you have read the available [Documentation](https://github.com/dmitriy-borisov/postmessage-transport#readme).

Before you ask a question, it is best to search for existing [Issues](https://github.com/dmitriy-borisov/postmessage-transport/issues) that might help you. In case you have found a suitable issue and still need clarification, you can write your question in this issue. It is also advisable to search the internet for answers first.

If you then still feel the need to ask a question and need clarification, we recommend the following:

- Open an [Issue](https://github.com/dmitriy-borisov/postmessage-transport/issues/new).
- Provide as much context as you can about what you're running into.
- Provide project and platform versions (nodejs, npm, etc), depending on what seems relevant.

We will then try to answer as soon as possible.

### Your First Code Contribution

1. Fork the project.
2. Clone your fork.
3. Install dependencies: `pnpm install`
4. Create a new branch: `git checkout -b my-new-feature`
5. Make your changes.
6. Run tests: `pnpm test`
7. Commit your changes.
8. Push to the branch: `git push origin my-new-feature`
9. Submit a pull request.
